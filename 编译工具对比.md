## 一、核心架构差异

| | Vite | Webpack |
|---|---|---|
| **开发服务器** | 原生 ESM，不打包，浏览器直接加载模块 | 开发时也要完整打包，内存中生成 bundle |
| **打包器** | 生产用 Rollup | 自身就是打包器 |
| **配置语言** | ES Module（`import/export`） | CommonJS（`require/module.exports`） |
| **配置文件** | `vite.config.ts` | `webpack.config.js` |
| **生态成熟度** | 2020 年发布，生态快速增长中 | 2014 年发布，生态极其成熟 |

### 本质区别

```
Webpack 的思路（Bundle-based）：
  源码 → 完整打包成 bundle → 浏览器加载 bundle

Vite 的思路（Native ESM + Bundle）：
  Dev：源码 → 浏览器按需加载单个模块（不打包）
  Prod：源码 → Rollup 打包成优化产物
```

## 二、开发环境（Dev）对比

### 2.1 启动速度

| | Vite | Webpack |
|---|---|---|
| **冷启动** | 极快（毫秒级），不打包，按需编译 | 慢，必须先完整打包整个应用 |
| **大型项目** | 启动速度基本不变 | 启动时间随项目体积线性增长 |

**原因**：

```
Vite（1000 个模块的项目）：
  启动 → 0ms（不打包）
  浏览器请求首页 → 只编译首页用到的 ~20 个模块
  首屏可见 ≈ 300-500ms

Webpack（1000 个模块的项目）：
  启动 → 编译全部 1000 个模块 → 5-30s
  编译完成 → 浏览器加载 bundle
  首屏可见 ≈ 5-30s + 加载时间
```

### 2.2 热更新（HMR）

| | Vite | Webpack |
|---|---|---|
| **原理** | 精确到单个模块，只重新编译变化的模块 | 重新构建变化的模块及其依赖链 |
| **速度** | 无论项目多大，HMR 通常 < 100ms | 项目越大越慢，可能达到数秒 |
| **可靠性** | 偶尔需要手动刷新（边界情况） | 更成熟，边界情况处理更完善 |

```
Vite HMR 流程：
  修改 Button.tsx
  → 只重新编译 Button.tsx
  → 浏览器只替换 Button 模块
  → 50ms 完成

Webpack HMR 流程：
  修改 Button.tsx
  → 重新编译 Button.tsx + 所有引用它的模块
  → 生成更新后的 chunk
  → 浏览器替换整个 chunk
  → 200ms-3s 完成
```

### 2.3 开发中间件

```
# Vite 开发服务器能力（基于 connect/Polka）
- 原生 ESM 服务
- 内置 CSS / CSS Modules / PostCSS 支持
- 内置 TypeScript 转译（esbuild，不做类型检查）
- 内置 JSX 转译
- 内置静态文件服务
- API 代理（server.proxy）
- 支持插件扩展
```

```
# Webpack Dev Server 能力（基于 Express）
- Bundle 服务
- 需要 loader 处理 CSS/TS/JSX（css-loader, ts-loader, babel-loader）
- HMR
- API 代理（devServer.proxy）
- 支持插件扩展
```

## 三、生产环境（Production）对比

### 3.1 打包工具

| | Vite | Webpack |
|---|---|---|
| **底层** | Rollup | 自身 |
| **Tree-shaking** | Rollup 原生支持，更彻底 | Webpack 5 支持，但不如 Rollup 精确 |
| **代码分割** | Rollup 的静态分析，更优 | 动态 import + SplitChunksPlugin |
| **产物体积** | 通常更小 | 通常略大 |

### 3.2 构建速度

| | Vite | Webpack |
|---|---|---|
| **首次构建** | 较快（esbuild 预构建 + Rollup 打包） | 较慢 |
| **增量构建** | 无内置增量构建缓存 | Webpack 5 有持久化缓存 |
| **大型项目** | 比 Webpack 快 2-5 倍 | 随项目变大显著变慢 |

### 3.3 产物特点

```
Vite 产物结构（Rollup 输出）：
  dist/
    assets/
      index-[hash].js          # 入口
      vendor-[hash].js         # 第三方依赖
      index-[hash].css         # 样式
      logo-[hash].png          # 静态资源
    index.html

Webpack 产物结构：
  dist/
    static/
      js/
        main.[hash].js         # 入口 chunk
        vendors.[hash].js      # 第三方依赖 chunk
      css/
        main.[hash].css        # 样式
      media/
        logo.[hash].png        # 静态资源
    index.html
```

### 3.4 兼容性

| | Vite | Webpack |
|---|---|---|
| **目标浏览器** | 默认支持原生 ESM 的浏览器（ES2020+） | 可配置支持 IE11 等老旧浏览器 |
| **降级方案** | `@vitejs/plugin-legacy`（用 Babel 降级） | 原生支持各种 target 配置 |
| **Node 版本** | Node 18+ | Node 12+（Webpack 5） |

## 四、优劣势总结

### Vite

**优势：**
1. 开发体验极佳 — 启动快、HMR 快
2. 配置简单 — 开箱即用，默认配置覆盖 90% 场景
3. 产物更优 — Rollup 的 tree-shaking 更彻底
4. 原生 TS/JSX/CSS Modules 支持，无需额外 loader
5. 插件系统简洁，兼容 Rollup 插件生态

**劣势：**
1. 生态不如 Webpack 成熟 — 部分复杂场景的插件可能缺失
2. 开发/生产行为不一致 — 开发用 esbuild，生产用 Rollup，偶有差异
3. CommonJS 依赖兼容问题 — 需要预构建（optimizeDeps），偶尔踩坑
4. 不支持老旧浏览器（IE11），需要额外插件
5. 某些 Webpack 特有功能（如 Module Federation）没有原生等价物

### Webpack

**优势：**
1. 生态最成熟 — 几乎所有场景都有现成解决方案
2. 开发/生产一致性 — 同一套打包逻辑
3. 高度可定制 — loader + plugin 可以处理任何场景
4. Module Federation — 微前端原生方案
5. 大量线上验证，稳定可靠

**劣势：**
1. 配置复杂 — 一个中型项目的 webpack config 可能 200+ 行
2. 开发启动慢 — 大型项目冷启动 10-60 秒
3. HMR 随项目变大变慢
4. 学习曲线陡峭 — loader/plugin/chunk/graph 概念多
5. 产物体积通常比 Rollup 大

### 选型建议

| 场景 | 推荐 | 原因 |
|---|---|---|
| 新项目（React/Vue） | **Vite** | 开发体验好，配置简单 |
| 老项目迁移 | 看情况 | 小项目可迁移，大项目迁移成本高 |
| 需要支持 IE11 | **Webpack** | Vite 需要额外插件，体验下降 |
| 微前端（qiankun） | 都可以 | 当前项目就是 Vite + qiankun |
| 微前端（Module Federation） | **Webpack** | 原生支持 |
| 大型企业应用 | **Webpack** | 生态成熟，边界情况处理完善 |
| 开源库/组件库 | **Vite**（库模式） | Rollup 产物更优 |

## 五、通用配置方式

### 5.1 Vite 通用配置

```ts
// vite.config.ts
import { defineConfig } from 'vite'

export default defineConfig({
  // --- 开发服务器 ---
  server: {
    port: 3000,
    open: true,                   // 自动打开浏览器
    cors: true,                   // 开启 CORS
    proxy: {                      // API 代理
      '/api': {
        target: 'http://localhost:8080',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api/, ''),
      },
    },
  },

  // --- 构建配置 ---
  build: {
    outDir: 'dist',
    sourcemap: true,              // 生成 sourcemap
    minify: 'esbuild',            // 压缩方式：esbuild | terser
    rollupOptions: {
      output: {
        manualChunks: {           // 手动分包
          vendor: ['react', 'react-dom'],
        },
      },
    },
  },

  // --- 路径别名 ---
  resolve: {
    alias: {
      '@': '/src',
    },
  },

  // --- CSS 配置 ---
  css: {
    modules: {
      localsConvention: 'camelCase',
    },
    preprocessorOptions: {
      scss: {
        additionalData: `@import "@/styles/variables.scss";`,
      },
    },
  },

  // --- 环境变量 ---
  // .env 文件中 VITE_ 前缀的变量自动暴露给客户端
  // 使用：import.meta.env.VITE_xxx
  envPrefix: 'VITE_',
})
```

### 5.2 Webpack 通用配置

```js
// webpack.config.js
const path = require('path')
const HtmlWebpackPlugin = require('html-webpack-plugin')
const MiniCssExtractPlugin = require('mini-css-extract-plugin')

module.exports = (env, argv) => {
  const isDev = argv.mode === 'development'

  return {
    // --- 入口 ---
    entry: './src/index.tsx',

    // --- 输出 ---
    output: {
      path: path.resolve(__dirname, 'dist'),
      filename: isDev ? '[name].js' : '[name].[contenthash:8].js',
      clean: true,
    },

    // --- 开发服务器 ---
    devServer: {
      port: 3000,
      hot: true,
      open: true,
      historyApiFallback: true,   // SPA 路由
      proxy: {
        '/api': {
          target: 'http://localhost:8080',
          changeOrigin: true,
        },
      },
    },

    // --- 模块处理规则（loader 从右到左执行）---
    module: {
      rules: [
        // TypeScript / JSX
        {
          test: /\.(ts|tsx)$/,
          exclude: /node_modules/,
          use: 'babel-loader',    // 或 ts-loader
        },
        // CSS
        {
          test: /\.css$/,
          use: [
            isDev ? 'style-loader' : MiniCssExtractPlugin.loader,
            'css-loader',
            'postcss-loader',
          ],
        },
        // SCSS
        {
          test: /\.scss$/,
          use: [
            isDev ? 'style-loader' : MiniCssExtractPlugin.loader,
            'css-loader',
            'postcss-loader',
            'sass-loader',
          ],
        },
        // 静态资源
        {
          test: /\.(png|jpg|svg|gif)$/,
          type: 'asset/resource',
        },
      ],
    },

    // --- 插件 ---
    plugins: [
      new HtmlWebpackPlugin({
        template: './public/index.html',
      }),
      ...(!isDev ? [new MiniCssExtractPlugin({
        filename: '[name].[contenthash:8].css',
      })] : []),
    ],

    // --- 路径别名 ---
    resolve: {
      alias: {
        '@': path.resolve(__dirname, 'src'),
      },
      extensions: ['.ts', '.tsx', '.js', '.jsx', '.json'],
    },

    // --- 优化 ---
    optimization: {
      splitChunks: {
        chunks: 'all',
        cacheGroups: {
          vendor: {
            test: /[\\/]node_modules[\\/]/,
            name: 'vendors',
            chunks: 'all',
          },
        },
      },
    },

    // --- SourceMap ---
    devtool: isDev ? 'eval-cheap-module-source-map' : 'source-map',
  }
}
```

### 5.3 配置复杂度对比

| 配置项 | Vite | Webpack |
|---|---|---|
| TS 支持 | 零配置 | 需要 `ts-loader` 或 `babel-loader` + 预设 |
| JSX 支持 | 零配置（插件一行） | 需要 `babel-loader` + `@babel/preset-react` |
| CSS Modules | 零配置（`.module.css`） | 需要 `css-loader` 配置 `modules: true` |
| SCSS | 安装 `sass` 即可 | 需要 `sass-loader` + `css-loader` + `style-loader` |
| 静态资源 | 零配置 | 需要 `asset/resource` 类型配置 |
| 环境变量 | `.env` 文件 + `VITE_` 前缀 | `DefinePlugin` 或 `dotenv-webpack` |
| 路径别名 | `resolve.alias` | `resolve.alias` + `resolve.extensions` |
| API 代理 | `server.proxy` | `devServer.proxy` |
| 代码分割 | Rollup 自动 + `manualChunks` | `SplitChunksPlugin` 配置 |

## 六、React 项目配置

### 6.1 Vite + React

```bash
# 创建项目（CLI 一行搞定）
npm create vite@latest my-app -- --template react-ts
```

```ts
// vite.config.ts — 典型 React 项目配置
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [
    react({
      // 使用 babel（默认用 esbuild 转译 JSX）
      // 如果需要 emotion / styled-components 等 babel 插件时开启
      babel: {
        plugins: [['@babel/plugin-transform-react-jsx', { runtime: 'automatic' }]],
      },
    }),
  ],
  server: {
    port: 3000,
    proxy: {
      '/api': 'http://localhost:8080',
    },
  },
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          'react-vendor': ['react', 'react-dom', 'react-router-dom'],
        },
      },
    },
  },
})
```

### 6.2 Webpack + React（CRA / 手动配置）

```bash
# CRA 创建（封装了 webpack，零配置）
npx create-react-app my-app --template typescript

# eject 暴露 webpack 配置（不可逆！）
npm run eject
```

```js
// webpack.config.js — React 关键配置
module.exports = {
  module: {
    rules: [
      {
        test: /\.(js|jsx|ts|tsx)$/,
        exclude: /node_modules/,
        use: {
          loader: 'babel-loader',
          options: {
            presets: [
              '@babel/preset-env',
              ['@babel/preset-react', { runtime: 'automatic' }],
              '@babel/preset-typescript',
            ],
          },
        },
      },
    ],
  },
  resolve: {
    extensions: ['.js', '.jsx', '.ts', '.tsx'],
  },
}
```

### 6.3 React 配置差异总结

| | Vite | Webpack (CRA) |
|---|---|---|
| 创建命令 | `npm create vite` | `npx create-react-app` |
| JSX 转译 | esbuild（默认，快）或 Babel | Babel |
| Fast Refresh | `@vitejs/plugin-react` 内置 | `react-refresh` webpack 插件 |
| 自定义配置 | `vite.config.ts` 直接改 | 需要 eject 或用 `craco` / `react-app-rewired` |
| 环境变量 | `VITE_` 前缀 | `REACT_APP_` 前缀 |

## 七、Vue 项目配置

### 6.1 Vite + Vue

```bash
# 创建项目（官方推荐方式）
npm create vue@latest    # 使用 create-vue（官方脚手架）
# 或
npm create vite@latest my-app -- --template vue-ts
```

```ts
// vite.config.ts — 典型 Vue 项目配置
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  plugins: [vue()],
  server: {
    port: 3000,
  },
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          'vue-vendor': ['vue', 'vue-router', 'pinia'],
        },
      },
    },
  },
})
```

### 6.2 Webpack + Vue（Vue CLI）

```bash
# Vue CLI 创建（基于 webpack）
npm install -g @vue/cli
vue create my-app
```

```js
// vue.config.js — Vue CLI 的 webpack 配置层
module.exports = {
  configureWebpack: {
    // 直接写 webpack 配置
  },
  chainWebpack: (config) => {
    // 链式 API 修改 webpack 配置
    config.resolve.alias.set('@', path.resolve('src'))
  },
  devServer: {
    port: 3000,
    proxy: {
      '/api': 'http://localhost:8080',
    },
  },
}
```

### 6.3 Vue 配置差异总结

| | Vite | Webpack (Vue CLI) |
|---|---|---|
| 创建命令 | `npm create vue@latest` | `vue create` |
| 官方推荐 | **Vite**（Vue 3 官方推荐） | 已进入维护模式 |
| SFC 编译 | `@vitejs/plugin-vue` | `vue-loader` |
| Vue CLI 状态 | — | **已不再积极维护** |
| 自定义配置 | `vite.config.ts` | `vue.config.js` |

## 八、CLI 脚手架配置是否够用？

### 结论

**大多数项目，CLI 生成的配置已经足够**，不需要手动改配置文件。

### 什么时候够用？

| 场景 | CLI 是否够用 | 说明 |
|---|---|---|
| 标准 SPA 开发 | **够用** | 路由、状态管理、API 调用、样式处理 |
| TypeScript | **够用** | Vite/CLI 都已内置支持 |
| CSS Modules / SCSS | **够用** | 安装预处理器即可 |
| 代理后端 API | **够用** | `server.proxy` 配置几行 |
| 路径别名 | **够用** | 一行配置 |
| 环境变量 | **够用** | `.env` 文件 |

### 什么时候需要自定义配置？

| 场景 | 需要自定义 | 具体改什么 |
|---|---|---|
| 微前端（qiankun） | 需要 | `vite-plugin-qiankun` 插件 + 生命周期导出 |
| 多入口 / MPA | 需要 | `rollupOptions.input` |
| 组件库 / SDK 开发 | 需要 | `build.lib` 库模式 |
| 自定义 HTML 模板 | 需要 | `htmlWebpackPlugin` 或 `transformIndexHtml` |
| SVG 图标系统 | 需要 | `svgr` 插件 |
| Webpack Module Federation | 需要 | Webpack 配置 `ModuleFederationPlugin` |
| 国际化按需加载 | 可能需要 | 自定义 chunk 策略 |
| 构建性能优化（超大项目） | 可能需要 | 持久化缓存、分包策略 |
| 旧浏览器兼容 | 可能需要 | `@vitejs/plugin-legacy` |

### 推荐的工作流

```
1. 用 CLI 脚手架创建项目（不要从零手写配置）
2. 默认配置开发，遇到问题再改
3. 需要自定义时，只改必要的部分
4. 不要过度配置 — 能跑就别动
```

### 各框架推荐的脚手架

| 框架 | 推荐脚手架 | 底层 |
|---|---|---|
| React | `npm create vite@latest` | Vite |
| React（旧） | `create-react-app` | Webpack（已不推荐） |
| React（全功能） | Next.js | Webpack/Turbopack |
| Vue | `npm create vue@latest` | Vite |
| Vue（旧） | `vue create` | Webpack（已维护模式） |
| Vue（全功能） | Nuxt | Vite |

---

## 总结一句话

> **新项目选 Vite，老项目用 Webpack，CLI 脚手架覆盖 90% 场景，只在微前端/库开发/特殊需求时手动配置。**
