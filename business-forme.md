## 柏视科技冲刺计划（5-7天详细方案）

你既然最想去柏视科技，那就全力冲刺。以下是你每天应该做什么：

### Day 1-2：NestJS 核心体系恢复

你已经用过 NestJS，不需要从零学，重点是**唤醒记忆 + 补新特性**。

**Day 1：模块体系 + 请求处理管道**
- [ ] 复习模块（Module）、控制器（Controller）、服务（Service）的组织方式
- [ ] 复习依赖注入（DI）机制：`@Injectable()`、自定义 Provider、工厂 Provider
- [ ] 复习请求生命周期：Middleware → Guard → Interceptor → Pipe → Controller → Service
- [ ] 写一个简单的 CRUD 模块（比如用户管理），找回手感
- [ ] **面试高频题**：NestJS 的依赖注入是怎么实现的？和 Angular 的 DI 有什么异同？

**Day 2：数据库 + 认证授权**
- [ ] 复习 TypeORM：Entity 定义、Repository 模式、关系映射（OneToOne/OneToMany/ManyToMany）
- [ ] 复习事务处理：`@Transaction()` 装饰器或 `QueryRunner`，了解嵌套事务
- [ ] 复习 JWT 认证：`@nestjs/jwt` + `@nestjs/passport` 的完整流程
- [ ] 复习 RBAC 实现：你做过的用户-角色-权限模型，整理成面试话术
- [ ] **实操**：用 NestJS + TypeORM 搭一个带 RBAC 的用户认证模块（和你 AIOT 项目类似）
- [ ] **面试高频题**：JWT 和 Session 的区别？RBAC 和 ABAC 的区别？Token 刷新策略？

### Day 3：NestJS 进阶 + 微服务概念

- [ ] 复习 Guard（守卫）：`CanActivate` 接口、角色守卫、权限守卫
- [ ] 复习 Interceptor（拦截器）：响应转换、日志记录、缓存
- [ ] 复习 Pipe（管道）：验证（ValidationPipe）、Transform
- [ ] 复习 DTO + class-validator：请求参数验证
- [ ] 了解 NestJS 微服务模式：`@MessagePattern`、Transport 层（TCP/Redis/RMQ），不需要深入，但要能讲清楚概念
- [ ] **面试高频题**：NestJS 的 Guard 和 Middleware 的区别？如何做接口级别的权限控制？

### Day 4：数据库设计 + RESTful API

- [ ] **数据库设计练习**：针对医疗场景，设计几个核心表结构
  - 患者表（patients）：姓名、身份证、手机号、病历号
  - 科室表（departments）：科室名称、科室类型
  - 医生表（doctors）：关联科室、职称
  - 挂号订单表（appointments）：关联患者、医生、科室、时间、状态
  - 医疗记录表（medical_records）：关联患者、医生、诊断结果、处方
- [ ] 复习 RESTful API 设计规范：资源命名、HTTP 方法、状态码、分页、过滤
- [ ] **面试高频题**：如何设计一个支持复杂查询的 API（多条件筛选 + 分页 + 排序）？
- [ ] 了解基本的事务隔离级别（Read Committed、Repeatable Read）

### Day 5：医疗业务 + 系统集成概念

- [ ] **医疗业务快速了解**（面试前至少能说出概念）：
  - HIS（医院信息系统）：挂号、收费、药房管理等核心业务
  - LIS（实验室信息系统）：检验报告管理
  - PACS（影像归档与通信系统）：CT/MRI 等影像存储和查看
  - EMR（电子病历）：医生书写的病历记录
- [ ] **医疗数据标准**（加分项，了解即可）：
  - HL7：医疗数据交换标准，知道是什么就行
  - FHIR：HL7 的新一代标准，RESTful 风格
  - DICOM：医学影像标准格式
- [ ] **医疗安全规范**（了解概念即可）：
  - 等保三级：国内医疗系统安全等级要求
  - HIPAA：美国医疗隐私法规，了解一下思路
  - 核心原则：数据加密存储、传输加密、访问审计、最小权限
- [ ] **面试话术**：结合你的医疗智能客服项目，说明你对医疗数据隐私的理解（内网部署、数据不外传、操作审计）

### Day 6：项目经验整合 + 面试话术
